## App 描述

### 简短描述（170 字符以内）
感觉日子嗖地就过去？TSD 帮你为某些瞬间划重点——三个月后，你能讲出 5-10 个鲜明故事。

### 完整描述（4000 字符以内）

日子嗖地就过去了。一周，一月，一年——回头看，又想不起具体发生过什么。

这不是你的问题。大脑把重复的日子压缩成了一团。生活需要被划重点。

TSD 帮你做三件事：

**留住**——看到什么让你有感触，Mark 一张照片、选个心情、说一句话。5 秒就够。不需要每天写日记，不需要打卡，不需要坚持。

**讲述**——每周末，从这周的瞬间里挑 2-3 个，给这一周起个名字。TSD 帮你串成故事。这就是"这一周讲的是什么"。

**回望**——每月、每季度，TSD 把你的瞬间编译成可讲述的故事。三个月后，你能不费力地讲出 5-10 个鲜明瞬间——发生了什么、和谁有关、为什么重要。

■ 关于人生格子

TSD 把你的一生画成 4160 个格子——80 年，每周一格。已度过的格子被点亮，当前这一周在脉冲。每 Mark 一个瞬间，对应的格子变亮。这不是为了制造焦虑，而是让你看见：这一周，值得被记住。

■ 关于旷野

你的瞬间会生长出一片旷野——青草是日常，花朵是鲜明记忆，树木是长期关系。可以按时间、人物、主题三个镜头浏览，也可以缩放到一生看全景。

■ 隐私

你的记忆归你所有。所有数据默认存储在你的设备上，不上传任何服务器。你可以随时导出 JSON 备份，或彻底清除所有数据。

■ 适合谁

- 30-50 岁，感觉时间过得越来越快
- 不写日记，但愿意每天花 1-5 分钟
- 重视家人、朋友和真实生活
- 想把经历转化为以后讲得出来的记忆

■ 不适合谁

- 想要社交媒体 Feed 的人（TSD 不做公开分享）
- 想要 AI 替自己写日记的人（TSD 只帮你编辑，不替你决定意义）
- 想要习惯打卡 streak 的人（TSD 不惩罚遗漏）

## 隐私清单（Privacy Manifest / PrivacyInfo.xcprivacy）

### 数据收集类型
TSD **不收集任何用户数据**。所有记忆数据存储在用户设备本地（localStorage / CoreData）。

### 使用的 API
- **Photos**：仅在用户主动选择照片时访问（NSPhotoLibraryUsageDescription）
- **Camera**：用户主动拍照时（NSCameraUsageDescription）
- **Notifications**：可选的晚间扫描提醒（用户授权后）

### 隐私政策 URL
需注册独立域名托管隐私政策页面（如 tsd.app/privacy）。
当前 Demo 的隐私政策内置于 App 设置页。

### 权限说明文案（iOS）

**NSPhotoLibraryUsageDescription**:
"TSD 需要访问你的照片库，让你选择要 Mark 的瞬间照片。我们不会读取或上传你的其他照片。"

**NSCameraUsageDescription**:
"TSD 需要使用相机，让你直接拍摄要留住的瞬间。照片仅保存在你的设备上。"

**NSPhotoLibraryAddUsageDescription**:
"TSD 需要保存你生成的分享故事卡到相册。"

## 审核 Note（给 App Review 团队）

TSD 是一个记忆记录和回望工具。App 内不包含任何账号系统、社交功能、内购或广告。

审核路径：
1. 打开 App → 完成 onboarding（5 屏）
2. 点底部 ＋ Mark 一个瞬间（选照片 + 心情 + 一句话）
3. 回到首页看到今晚扫描卡片
4. 切到底部"旷野"tab → 看到 SVG 旷野
5. 切到"原料"tab → 切换列表/照片墙/地图三种视图
6. 设置 → 关于 TSD → 查看隐私政策

Demo 模式：设置页底部"看示例数据"可查看三个月完整数据演示。

### 构建 Demo → 生产 App 的差距
1. **PWA → Capacitor/WebView**：用 Capacitor 包装当前 PWA 为 iOS App
2. **localStorage → CoreData**：迁移持久层到原生 CoreData
3. **FileReader → UIImagePickerController**：原生照片选取
4. **picsum → 用户真实照片**：demo 占位图移除
5. **Apple Push → 本地通知**：晚间扫描用 UILocalNotification
6. **App Store 图标 1024px**：从当前 512px 放大或重新设计
