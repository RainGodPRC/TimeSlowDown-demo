## 决策矩阵

### 如果 A 组 moment_saved 显著高于 B 组
→ 额外锚点（每日一词/意外关联/稀缺标签/社会认同/微承诺）**有效**
→ 保留全部 20 个锚点

### 如果 A 组和 B 组 moment_saved 无显著差异
→ 额外锚点**不影响核心行为**
→ 砍掉 B 组不显示的那 5 个锚点（简化 UI，减少信息过载）

### 如果 B 组 upgrade_completed 高于 A 组
→ 信息过载假设成立（A 组锚点太多反而降低转化）
→ 砍掉更多锚点，进一步精简

### 如果 voice_started 计数为 0
→ 语音 Mark 不被使用
→ 可能原因：权限恐惧 / 不知道怎么用 / 不需要
→ 考虑加引导或移除

## 分析工具链验证（Demo 模式）

在 Demo 模式下，可以模拟使用产生 analytics 数据，然后导出验证工具链：

1. 打开设置 → "看示例数据"
2. 在讲述首页点击各种功能（今晚扫描、说点什么、周末章节）
3. 设置 → 锚点效果数据 → 查看数据是否记录
4. 导出 JSON → 检查格式是否正确

**注意**：Demo 模式下 track() 可能不触发（某些锚点条件依赖 empty mode）。建议在 empty 模式下测试。
